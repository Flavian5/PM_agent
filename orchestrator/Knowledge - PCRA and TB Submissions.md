# PCRA and Treasury Board Submissions

## Project Complexity and Risk Assessment (PCRA)

### What is PCRA?
The PCRA is a Treasury Board-mandated artifact that determines if SSC has the authority to manage a project without Treasury Board oversight.

### PCRA Levels

| Level | Description | Authority Required |
|-------|-------------|-------------------|
| **Level 1** | Low complexity | SSC can approve internally |
| **Level 2** | Moderate complexity | SSC can approve internally |
| **Level 3** | High complexity | SSC can approve internally (current OPMC Class) |
| **Level 4** | Very high complexity | Requires Treasury Board approval |

### SSC's Current Authority
- **OPMC Class**: 3
- Any project with PCRA Level 4 must submit a TB Submission before seeking Gate 2 approval

---

## When PCRA is Required

### SSC-Led Projects
- PCRA required at Gate 0 (estimate for planning)
- PCRA required at Gate 1 (refined)
- Formal PCRA required for projects with total cost (Gate 2-5) over **$2.5M** (includes EBP, accommodations, corporate overhead)
- Must be updated when scope, cost, or schedule changes by **>= 10%**

### Customer-Led Projects
- SSC does NOT prepare PCRA
- Customer provides their PCRA to confirm their project authority
- Customer's PCRA applies to entire project including SSC component

---

## TB Submission Requirements

### When TB Submission is Required

1. **PCRA Level 4** - Exceeds SSC's OPMC Class 3 authority
2. **Project over $10M** - Requires Concept Case submission to TBS
3. **Requested by TBS** - As directed by Treasury Board

### Types of TB Submissions

| Type | Purpose | Cost Requirement |
|------|---------|------------------|
| **Project Approval (PA)** | Seek approval for project | Substantive for implementation |
| **Expenditure Authority (EA)** | Access funding | Substantive for activities |
| **Financial Authority (FA)** | Access Fiscal Framework funds | Substantive for stage |
| **Contract Entry Authority** | Enter/amend contracts exceeding SSC limit | N/A |

### TB Submission Timeline
- **Duration**: Can take over 6 months from initiation to approval
- **Factor into schedule**: Appropriate time must be planned

---

## PCRA 4 Constraints

### Key Restrictions
- **Cannot employ iterative gating** until post Gate 3 approval
- Must seek TB approval before Gate 2
- TBSU briefs Ministers on total project costs

### Process for PCRA 4 Projects
1. Complete PCRA at Gate 0/1
2. Consult with TBSU to determine if TB Submission needed
3. If required, prepare TB Submission with:
   - Sponsor Appointment Letter
   - Project Brief
   - Additional artifacts
4. Submit to TBS for review
5. Wait for TBS acknowledgment before proceeding to Gate 2

---

## PCRA Update Triggers

### When to Update PCRA
- Changes to scope >= 10%
- Changes to cost >= 10%
- Changes to schedule >= 10%
- Significant changes to expected benefits

### Process for Updates
1. Consult with PRO Lead
2. Update PCRA and supporting artifacts
3. Depending on PCRA level, may need TBS endorsement

## PCRA Scoring Factors

The PCRA evaluates:
- Project complexity
- Risk factors
- Strategic alignment
- Resource requirements
- Stakeholder complexity
- Technical complexity
- External dependencies

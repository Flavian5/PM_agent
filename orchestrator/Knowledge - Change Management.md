# Change Management

## Project Change Requests (PCR)

### When is a PCR Required?
A Project Change Request is required when changes affect:
- Scope
- Schedule
- Cost
- Benefits
- Project baseline

---

## Approval Thresholds

### Project Manager / Sponsor / Steering Committee Approval
- Technical solution or documentation changes that do NOT affect cost, scope, schedule, or benefits
- Changes to tasks or resources that do NOT affect the critical path

### PMB (Project Management Board) Approval Required
- Changes to project scope, schedule, cost, or expected benefits
- Use of project's contingency fund
- Movement of funds between sources or fiscal years
- Movement of funds between cost categories:
  - Salary ↔ O&M
  - Capital Salary ↔ Capital O&M
- Cancellations
- Conversions to activities or PMLIs
- Project name changes
- Placing projects on hold

### FIMB + PMB Approval (SSC-Led Projects)
- Any increase to overall project costs (conditional PMB approval, closed once FIMB approves)

### Treasury Board Approval (PCRA 4 or as directed by TBS)
- Total estimated cost exceeds TB-approved expenditure limits
- Significant change to anticipated immediate outcomes and benefits
- Significant change to project baseline
- Any change to project end date

---

## 10% Rule

### Trigger
If changes to scope, schedule, cost, or benefits are **>= 10%**:
- Project Manager must prepare an updated PCRA
- Supporting project artifacts must be updated (e.g., Business Case)
- Depending on PCRA level, may need TBS endorsement

---

## Change Request Process

### Step-by-Step
1. **Identify change** - Assess impact on scope, schedule, cost, benefits
2. **Determine approval level** - PM/Sponsor, PMB, or TBS
3. **Prepare PCR** - Document change, rationale, impact
4. **Obtain approvals** - Per threshold table above
5. **Update artifacts** - PCRA, Business Case, etc. if >= 10%
6. **Communicate** - Notify stakeholders of approved change

---

## Customer-Led Project Changes

### Special Requirements
- **Client Executive must review ALL PCRs** before presentation to PMB
- Changes apply to SSC component only
- Customer authority governs overall project

---

## Governance Review and Endorsement

### Prerequisites for Gate Review
- Documents reviewed by Project Team, Sponsor, SISD, stakeholders
- Costing evidence completed and available to CMA
- Actuals obtained from FinGov and included in financial tables
- Financials aligned across all artifacts

### Review Process
1. Provide links to documents to PRO Lead, CMA, FinGov
2. Reviewers provide feedback and recommended changes
3. Address feedback until reviewers are satisfied
4. Obtain confirmation of endorsement
5. Proceed to gate signing

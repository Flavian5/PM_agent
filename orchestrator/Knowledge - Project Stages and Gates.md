# Project Stages and Gates

## SSC Project Lifecycle Overview

### SSC-Led Projects (6 Stages)
1. **Stage 0: Intake and Concept** → Gate 0
2. **Stage 1: Investment (SSC-led Only)** → Gate 1
3. **Stage 2: Preliminary Planning** → Gate 2
4. **Stage 3: Detailed Planning** → Gate 3
5. **Stage 4: Execution** → Gate 4
6. **Stage 5: Deployment and Closeout** → Gate 5

### Customer-Led Projects (5 Stages)
1. **Stage 0: Intake and Concept** → Gate 0
2. **Stage 2: Preliminary Planning** → Gate 2 (no Stage 1)
3. **Stage 3: Detailed Planning** → Gate 3
4. **Stage 4: Execution** → Gate 4
5. **Stage 5: Deployment and Closeout** → Gate 5

---

## Stage Descriptions

### Stage 0: Intake and Concept
- Assess if proposal meets project criteria using Project Evaluation Tool (PET)
- Develop high-level scope, schedule, and cost estimate (ROM +/-100%)
- Create Investment Proposal (SSC-Led) or use PCW (Customer-Led)
- Complete Concept Case for projects over $10M
- Present to 5P Board for Gate 0 approval

### Stage 1: Investment (SSC-Led Only)
- Establish project governance structure
- Refine scope and develop options
- Define outcomes and benefits
- Develop Business Case
- Present to FIMB for Gate 1 approval

### Stage 2: Preliminary Planning
- Develop comprehensive business and technical requirements
- Create Conceptual Design Document (CDD)
- Develop procurement plan and strategy
- Complete Privacy Risk Checklist (PRC)
- Understand security requirements
- Refine cost estimate to Substantive (+/-10%) for next stage
- Present to PMB for Gate 2 approval

### Stage 3: Detailed Planning
- Complete Solution Integration Document (SID)
- Draft Test Strategy and Plan
- Refine schedule and Work Breakdown Structure (WBS)
- Begin drafting Concept of Operations
- Plan for security assessments
- Present to PMB for Gate 3 approval

### Stage 4: Execution
- Execute procurement strategy
- Build and configure solution
- Develop transition plans
- Obtain Authority to Operate (ATO)
- Deliver training and documentation
- Transfer assets to Service Lines
- Present to PMB for Gate 4 approval

### Stage 5: Deployment and Closeout
- Deploy solution to production
- Manage and close contracts
- Record final project actuals
- Complete Closeout Report
- Present to PMB for Gate 5 approval
- Release project team

---

## Iterative Gating

### When to Use Iterative Gating
- **Scenario 1**: Scope not fully known - can estimate cost/schedule for some work while refining remaining
- **Scenario 2**: Schedule has significant external dependencies

### Key Constraints
- PCRA Level 4 projects CANNOT employ iterative gating until post Gate 3
- Projects can have multiple iterations of Gates 2, 3, and 4
